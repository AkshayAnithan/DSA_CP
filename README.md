## 🧭 My Solving Workflow

```mermaid
flowchart LR
    A[Pick next problem] --> B[Read 2x, restate in own words]
    B --> C[Brute force first — always]
    C --> D[Identify pattern / optimize]
    D --> E[Code in C++ or C#]
    E --> F[Test edge cases by hand]
    F --> G{Passed?}
    G -- Yes --> H[Write 2-line note above solution]
    G -- No, stuck 30 min --> I[Read editorial / hint]
    I --> J[Re-implement from scratch, closed tab]
    J --> H
    H --> K[Mark solved in tracker]
    K --> L[Add to revisit queue if hard]
```

### Rules of engagement

- ⏳ **30-minute rule** — stuck for 30 min? Peek at a hint. Stuck 60? Read the editorial, then re-implement from scratch.
- ✍️ **Note-above-code** — every solution starts with a 2-line comment: _pattern used_ + _time/space_.
- 🔁 **Revisit hards** — every 🔴 goes into a "redo in 3 days" queue.
- 🌐 **Both languages** — solve in C++ first, port tricky ones to C# for interview flexibility.
